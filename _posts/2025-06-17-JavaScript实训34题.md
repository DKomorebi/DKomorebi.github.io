## 第一部分：基础逻辑与算法 (1-10题)

### 1. 成绩判定
- **核心代码：** `switch(true)` 配合 `case score >= 90`。
- **讲解：** “我特意没用普通的 `switch(score)`，因为成绩是一个区间。通过 `switch(true)`，我可以在 `case` 里直接写**布尔表达式**。这样代码结构比 `if...else if` 更加扁平化，易于阅读。”

### 5. 递归阶乘
- **核心方法：** `factorial(n)` 递归调用。
- **关键行：** `return n * factorial(n - 1)`。
- **讲解：** “这里利用了**递归思想**。函数自己调用自己，每次把规模缩小。我设置了 `n === 1` 作为递归的**终止条件（Base Case）**，防止死循环。逻辑上，10的阶乘就是 10 乘以‘9的阶乘结果’。”

### 7. 排序算法
- **核心逻辑：** 双重 `for` 循环与变量交换。
- **关键行：** `const temp = arr[j]; arr[j] = arr[j+1]; arr[j+1] = temp;`
- **讲解：** “**冒泡排序**的核心是‘两两比较’，外层循环控制轮数，内层循环 `arr.length - i - 1` 确保每轮排好的最大值不再参与比较。交换位置时我用了一个 `temp` 中间变量。”

### 10. 学生数据处理
- **核心方法：** `toFixed(1)` 保留小数。
- **关键行：** `totalScore += students[i].score`。
- **讲解：** “计算平均分时，我先通过 `for` 循环累加 `score` 属性。最后返回时使用了 `toFixed(1)` 方法，它能确保结果是**字符串格式**且保留一位小数，非常适合展示成绩。”

## 第二部分：DOM 交互与 BOM 接口 (11-20题)

### 13. 回到顶部
- **核心接口：** `window.scrollY` 与 `window.scrollTo`。
- **配置项：** `behavior: 'smooth'`。
- **讲解：** “我通过 `window.addEventListener('scroll', ...)` 监听滚动。利用 `window.scrollY` 获取当前滚动的像素值。在点击事件里，我调用了 `scrollTo` 方法，并传入了对象参数 `{ top: 0, behavior: 'smooth' }`，实现了**原生平滑滚动**，不需要额外写计时器。”

### 14. 全选功能
- **核心属性：** `.checked` (布尔值)。
- **逻辑点：** 数组 `every` 思想（或手动循环判断）。
- **讲解：** “全选框的 `change` 事件直接同步给所有子项。而子项的 `change` 事件里，我写了一个逻辑：只有当 `item1.checked && item2.checked && item3.checked` 全为真时，主选框才会被勾选。这体现了逻辑中的**‘全真才真’**。”

### 16. 轮播图切换
- **核心操作：** `classList.add('active')` 与 `remove`。
- **讲解：** “我定义了一个全局变量 `currentIndex`。点击右按钮时执行 `currentIndex++`，并通过取模运算或 `if` 判断实现**首尾相连（循环播放）**。显示图片时，我先用 `forEach` 移除所有图片的 `active` 类，再给当前索引项添加，这是典型的**排他操作**。”

### 20. 事件委托
- **核心属性：** `event.target`。
- **关键行：** `if (event.target.tagName.toLowerCase() === 'li')`。
- **讲解：** “这是性能优化的重点。我没有给每一个 `li` 绑事件，而是绑在父级 `ul` 上。通过 `event.target` 捕捉点击源，并用 `tagName` 判断是否为 `li`。这种**事件代理**技术在处理动态数据时非常有用。”

## 第三部分：高级应用与面向对象 (21-34题)

### 22. 成绩单转换 (Map)
- **核心方法：** `Array.prototype.map()`。
- **讲解：** “我使用了 `map` 方法来遍历二维数组。在 `map` 的回调函数里，我返回了一个全新的对象结构。这样做不仅代码量少，而且不会破坏原始的 `gradeBook` 数据，体现了**函数式编程**的理念。”

### 23. 优先级队列
- **核心方法：** `Array.prototype.splice(index, 0, item)`。
- **讲解：** “为了保证队列有序且不使用 `sort`，我在 `enqueue` 方法里做了一个手动查找。找到第一个优先级（数字）比当前消息大的位置，然后利用 `splice` 方法在该位置**原地插入**。这种方式插入后队列依然是排好序的。”

### 24. 入职纪念日 (Date)
- **核心方法：** `Date.now()` 或 `new Date()`。
- **逻辑点：** 闰年判断 `(year % 4 === 0 && year % 100 !== 0)`。
- **讲解：** “这题难点在于 **2月29日** 的边界处理。如果员工 2.29 入职，但今年不是闰年，我就通过 `new Date(year, 1, 28)` 强制把纪念日设为 2.28。最后通过减法得到两个日期的毫秒差，再除以一天的毫秒数 `(1000 * 60 * 60 * 24)` 换算成天数。”

### 32. 密码强度检测 (Regex)
- **核心正则：** `/\d/` (数字), `/[a-zA-Z]/` (字母), `/[!@#$%^&*]/` (特殊字符)。
- **方法：** `regex.test(string)`。
- **讲解：** “我在 `oninput` 事件里实时获取 `this.value`。使用了三个正则表达式的 `test` 方法来检测字符成分。最后根据组合结果（如长度加成分）动态切换提示区的 `className`，从而改变文字颜色。”

### 33. 面向对象分页器
- **核心方法：** `Array.prototype.slice(start, end)`。
- **原型定义：** `Pagination.prototype.render = function() { ... }`。
- **关键逻辑：** `const start = (page - 1) * this.pageSize;`。
- **讲解：** “我把分页逻辑封装在 `Pagination` 类中。`render` 方法的核心是利用 `slice` 切片。起始位置计算公式是 `(当前页 - 1) * 每页条数`。**这里我特别注意了 DOM 挂载顺序**：在 `init` 方法里，先创建按钮，再渲染数据并更新按钮状态，避免了因为按钮还没出现在 DOM 中而导致的报错。”

### 34. 简易购物车
- **核心数组方法：** `find()` 与 `filter()`。
- **讲解：** “在 `addToCart(id)` 中，我先用 `find` 方法检查购物车数组里是否已有该 `id` 的商品。如果有，直接修改 `count` 属性；如果没有，就从商品库里克隆一个对象推入。移除商品时则用了 `filter` 方法。每次数据变动后，我都会统一调用 `renderCart` 重新渲染 UI，确保了**单向数据流**的清晰。”
