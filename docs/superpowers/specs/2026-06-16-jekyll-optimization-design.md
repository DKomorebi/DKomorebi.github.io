# Jekyll 博客深度优化设计文档 (2026-06-16)

本方案旨在为基于 Jekyll 的 `shuimo` 主题博客提供全自动 WebP 转换、树状分类逻辑、折叠归档以及符合原主题美学的交互增强。

## 1. 核心需求目标
1. **图片全自动 WebP 化**：通过 GitHub Actions 实现“无感”转换。
2. **树状分类系统**：支持 `大类/中类/文章` 逻辑。
3. **同类邻近导航**：文章底部的“上一篇/下一篇”优先推荐同分类同等级的文章。
4. **折叠归档**：按年份/月份收纳，保持页面整洁。
5. **交互增强**：添加符合“水墨”风格的“回到顶部”按钮。

## 2. 详细设计方案

### 2.1 全自动 WebP 转换 (方案 A)
*   **转换引擎**：Node.js + `sharp`。
*   **自动化流程**：
    *   在 GitHub Actions 构建阶段，扫描 `assets/` 和 `images/` 目录。
    *   生成对应的 `.webp` 文件。
    *   **链接替换**：使用 `replace-in-file` 插件遍历生成的静态 HTML，将本地图片后缀替换为 `.webp`。
*   **本地开发**：本地预览仍显示原图，不影响创作体验。

### 2.2 树状分类逻辑 (方案 B)
*   **定义方式**：在文章 Front Matter 使用 `category_path: "分类/子分类/章节"`。
*   **分类页面 (`categories.html`)**：
    *   重写 Liquid 逻辑，将所有文章按 `category_path` 拆解并构建成树形嵌套对象。
    *   前端使用 `<details>` 或 CSS 交互实现层级展开。
*   **文章详情页导航**：
    *   修改 `post.html` 的上下文逻辑。
    *   **排序算法**：首先过滤出具有相同 `category_path` 的文章，按日期排序，寻找当前文章的前后邻居。若同级无邻居，则回退到全站邻居。

### 2.3 归档折叠与 UI 增强
*   **归档页面**：
    *   年份节点默认闭合（或仅展开本年）。
    *   使用 `shuimo` 主题现有的变量控制颜色，确保风格统一。
*   **回到顶部按钮**：
    *   **视觉设计**：采用水墨韵味的圆形图标，辅以轻微的渐变或毛笔触感边框。
    *   **功能**：平滑滚动至顶部，仅在页面下滑一定距离后显示。

## 3. 实施计划

### 第一阶段：自动化环境搭建
1.  创建 `package.json`，安装 `sharp`, `glob`, `jsdom`。
2.  编写 `scripts/convert-webp.js` 和 `scripts/post-build.js`。
3.  配置 `.github/workflows/deploy.yml`。

### 第二阶段：模板逻辑重构
1.  修改 `_layouts/categories.html`：实现树状解析逻辑。
2.  修改 `_layouts/archives.html`：添加折叠交互（HTML5 `<details>`）。
3.  修改 `_layouts/post.html`：重写邻近导航逻辑。

### 第三阶段：样式与交互优化
1.  在 `assets/css/shuimo.css` 中添加树状列表和折叠节点的样式。
2.  在 `assets/js/shuimo.js` 中添加“回到顶部”逻辑和滚动监听。

## 4. 成功标准
*   线上预览的所有本地图片后缀均为 `.webp`。
*   分类页面可以多级展开，且文章归属准确。
*   文章底部导航能精准指向“同一系列”的文章。
*   全站视觉风格高度统一，无违和感。
